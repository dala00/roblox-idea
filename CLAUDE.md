# CLAUDE.md

このリポジトリは **Robloxの新作ゲームのアイデアを出すための環境** です。ゲーム本体のコードは無く、アイデア生成・採点・レポート化のための Claude Code スキルとサブエージェントが本体です。

## このリポジトリでやること
ユーザーは「面白くて作れるRobloxゲームのアイデア」を探している。アイデアを大量に出し、独立した観点で採点・順位付けし、見やすいHTMLにまとめる。

## 主要コマンド
- `/roblox-idea-jam` … アイデアを並列生成→採点→順位→HTMLレポート出力（このリポジトリのメイン機能）
  - `/roblox-idea-jam <テーマ>` … テーマ縛り
  - `/roblox-idea-jam --count=N` … 本数指定
  - `/roblox-idea-jam --trends` … トレンドを強制再検索

## 構成
- `.claude/skills/roblox-idea-jam/SKILL.md` … オーケストレーション本体（手順はここが正）
- `.claude/skills/_shared/game-design-principles.md` … **ゲーム設計の観点（時間で腐らない不変の原則）の単一の正**。各スキル/エージェントがここを参照する（設計ビュー。9評価軸はこの採点ビュー）
- `.claude/agents/roblox-idea-generator.md` … アイデア発案サブエージェント（1起動=1案、JSONで返す）
- `.claude/agents/roblox-idea-critic.md` … 採点サブエージェント（生成役とは別人格、9観点×加重合計100点）
- `research/` … Robloxトレンド調査のキャッシュ（`roblox-trends-<日時>.md`、先頭に `調査日時:`）。**腐る市場データのみ**（不変の原則は上の principles.md 側）
- `ideas/` … 出力先。`ideas/<yyyymmddhhiiss>/` に `report.html` と `ideas.json`。**.gitignore済み**

## 評価軸（各1〜10 / 重み付き合計100点満点・9観点）
オリジナリティ・驚き(×1.4) / 雰囲気(×0.8) / 実際の面白さ＝コアループ(×1.4) / 間口＝若年・ゲーム苦手でも(×1.0) / マネタイズ(×1.2) / ソーシャル・招待性=ゲーム内(×0.9) / ソロ・少人数耐性(×0.8) / Roblox個人開発の実現性(×1.2) / クリップ映え・外部拡散力=TikTok/Shorts(×1.3)
（重み合計10.0 → 満点100。socialは「ゲーム内で盛り上がる/招待」、viralityは「ゲーム外で拡散・検索される」で別軸。）

## 守るべき方針・決定事項
- **実現可能性は「個人開発」前提**。ただし「個人でちょっと頑張れば作れる範囲」も可とする。派手すぎてスコープ破綻する案は減点。
- **多様性を最優先**。同一プロンプトの10並列は禁止。必ず別々のシードを割り当てて似た案の量産を防ぐ。
- **生成役と採点役は分離**したまま保つ（自己採点バイアス回避）。
- **「不変の設計原則」と「腐る市場トレンド」を分離する**。設計原則は `_shared/game-design-principles.md`（毎回参照・滅多に変えない）、市場データは `research/roblox-trends-*.md`（60日で再検索）。両者を混ぜない。新スキル（例: idea-to-spec）も原則mdを参照する。
- **トレンド接地のキャッシュ規則**: `research/` の最新mdが実行日から**60日以内**なら確認も再検索もせずそれを参照。**60日超／無い／`--trends`指定時は、すぐ検索せずまず「更新しますか？」とユーザーに確認**し、**yesのときだけ**web検索して調査日時付きで保存する。noなら既存mdを古くてもそのまま使う（無ければトレンド接地なしで続行）。
- 出力HTMLは**外部依存なしの単一ファイル**（CSSインライン、JS不要のバー表示）。日本語で書く。
- タイムスタンプは実行時の日時を使う（会話冒頭の currentDate / 実時間）。

## 補足
- git管理下（`main` ブランチ）。`.gitignore` で `ideas/`（出力）を除外、`research/` のトレンドmdはキャッシュとしてコミットして残す方針。
- Windows / PowerShell 環境。Roblox制作のノウハウは `roblox-gamedev:*` スキル群に詳しい（実現性評価の参考になる）。
- 応答は日本語で。
