# ゲーム設計の観点（Roblox個人開発 / 不変の原則）

このファイルは **時間で腐らない設計原則** の単一の正（source of truth）。
- `roblox-idea-jam`（生成・採点）と `idea-to-spec`（仕様化）など、各スキル/エージェントはここを参照する。
- **腐る市場データ（どのジャンルが伸び/飽和、何が稼いでるか）は `research/roblox-trends-*.md` 側**。混同しない。
- 採点の「9評価軸」はこの原則の**採点ビュー**（評価のための要約）。本ファイルは**設計ビュー**（作るための詳細）。両者は整合させる。
- 数値（モバイル比率等）は緩やかに動くので、引用時は出典と時点を添える。原則そのものは数値が動いても有効。

各観点に **採点軸との対応** を併記する（originality / vibe / fun / accessibility / monetization / social / solo / feasibility / first_session）。

> **設計の最重要点**: 「面白く“見える”か」ではなく「**本当に面白いか**」、そして「その面白さを**新規が1回目のプレイ中に離脱せず実際に体験できるか**」を最優先で問う。
> **集客は当面（最初は）広告が主**という前提（外部拡散=TikTok/Shorts等の発信力も、Roblox推薦アルゴリズムのオーガニック露出も当てにしない）。だからこそ **広告費で連れてきた一人を1回目で確実に楽しませ、翌日また来させられるか** が死活的——初回離脱は広告費がそのまま溶け、リテンションが立って初めて LTV>CAC が成立し課金につながる。これを担う採点軸が **first_session** = 〔初回体験で刺さるか〕。外部拡散もオーガニック露出も**採点しない**。

---

## 1. コアループ & 楽しさの核 〔→ fun, originality〕
- **主動詞(verb)を1つに絞る**。「このゲームで指がやり続ける1動作」は何か（撃つ/避ける/積む/集める/合わせる）。30秒で1周回って気持ちいいか。
- **手応え(juice/feedback)**。ヒット・成功・収穫の瞬間に音・光・数字・揺れの即時フィードバックがあるか。「気持ちよさ」はここで作られる。
- **スキルの床と天井**。誰でもできる(床)／極めると差が出る(天井)の両方があるか。床だけ＝すぐ飽き、天井だけ＝間口が死ぬ。
- **能動性の確保**。プレイヤーの操作が結果に効くか。オートバトラー/放置は「見てるだけ」に痩せやすい＝**運頼みの受け身**にしない一手を必ず入れる。
- **狙い設計**。エイムが簡単すぎると撃つ快感が消える（巨大ボス等）→ 面白さを「避け・弱点・タイミング・編成」に移す。

## 2. オンボーディング（最初の体験）〔→ first_session, accessibility, fun〕
- **最初の30秒で「楽しい」に到達**。チュートリアルでなく、即コアループの気持ちいい瞬間を1回踏ませる（セッション0設計）。
- **コアの面白さを“1回目のプレイ内”で必ず届ける**。間口（操作の易しさ）とは別問題で、「このゲームの本当の面白さ」を1回目で体験させられないと、潜在的にどれだけ面白くても伝わらず離脱する。導線・難度・テンポを“初見が離脱しない”側に倒す。これが first_session の核。
- **説明を読ませない**。アイコン・色・矢印・ハイライトで誘導。文字が読めない低年齢でも進める。
- **失敗してもすぐ復帰**。初回で詰ませない・即リトライ・死亡コストを軽く。
- タグライン（一言の売り）と実プレイ体験が一致しているか（看板に偽りなし）。

## 3. プログレッション & リテンション 〔→ first_session, fun, monetization, solo〕
- **3つの時間軸で何が変わるか**を必ず設計：最初の30秒 / 10分後 / 数時間〜数日後。
- **報酬スケジュールの重なり**。短期(毎ループ)・中期(セッション)・長期(コレクション/ランク)の目標が常に同時に走っているか。
- **「次の目標が常に見える」**。あと少しで届く人参を切らさない。数値インフレも「次が見える」ための道具。
- **再訪フック(D1/D7)**。放置進行・デイリー・期間限定・友達との約束など「また開く理由」。発見アルゴリズムは翌日再訪率を見る（→観点6）。
- **1回目のプレイ終わりに「明日また開く理由」が見えているか**＝ first_session のもう半分。初回でコアの面白さを体験させ（→観点2）、かつ翌日また遊びたくなる引き（次の目標/未解放/続きが気になる/友達との約束）を初回セッション内に提示できているか。ここまで届けば課金は時間の問題、という前提で設計する。
- **コンテンツ消費速度 vs 供給**。手作りコンテンツは必ず枯れる→**手続き的に組み合わせて深くする**設計に寄せる（→観点8）。

## 4. モバイル / プラットフォーム制約 〔→ accessibility, feasibility〕
- **第一ゲート＝「指1本で成立するか？」**（タップ/フリック/フリック+仮想スティック+1〜2ボタン）。キーボード前提・複雑入力は**遊び手の大多数を最初に捨てる**。
  - 根拠(時点付き): Robloxはモバイル約72〜80%、プレイ時間の8割超がスマホ、課金もモバイル発が約半分（2024〜2025）。出典は trends md の該当節参照。数値が動いても「モバイル主役」という前提は当面不変。
- **画面の向き**：横(ランドスケープ)が主流。3D移動系(探索/パルクール/対戦)は横一択。デフォルトSensorはUI崩れ事故が多く、意図的に固定するのが定石。
  - **縦持ちは少数派＝差別化の空き枠**。カメラを動かさず指1本で完結する系（放置/タップ育成・収集・リズム・たまり場）は縦と相性良。片手/ながらプレイ層に届く。
  - 振り分け: 3D移動系→横固定 / タップ完結の平面寄り→縦も検討。
- **短時間プレイ前提**。1セッションが細切れでも成立し、放置/オートで損しない作り。
- **通信遅延に強い設計**。シビアな物理同期・リアルタイム対人・精密リズム判定は遅延で崩れる→クライアント完結や見た目だけ物理風で逃がす（→観点8）。

## 5. 人数設計（ソロ⇔協力の両立）〔→ solo, social, feasibility〕
- **鉄則: 「ゼロ人でも面白い → 人がいるともっと面白い」の順で積む**。逆（人前提）は個人開発だと過疎で詰む。PvE/対ボスはこれを満たしやすい。
- 両立の必須4点：①敵HP/難度を**人数でスケール** ②**ソロ用の逃げ道**（難度緩和・ボット/NPC/仲間・蘇生多め） ③**途中参加・離脱OK** ④**報酬を人数で不利にしない**（ソロが損だと過疎が加速）。
- 「人が集まらないと成立しない系」は最難関。やるなら必ず上記＋サーバー定員を小さくして賑わい感を出す。

## 6. 集客と初回体験（広告で来た一人を離さない）〔→ first_session, social〕
- **本環境の前提＝集客は当面「広告」が主**。外部拡散（TikTok/YouTube Shorts）を伸ばせる発信力は無く、Roblox推薦アルゴリズムのオーガニック露出も当てにしない。よって設計の照準は「拡散する画」ではなく **「来た一人を1回目で確実に楽しませ、翌日また来させる」** こと一点に絞る。
- **広告経済の鉄則: 初回離脱＝広告費が溶ける**。有料で連れてきたプレイヤーが1回目でコアの面白さに到達せず去ると、その獲得コストは丸損。リテンション（D1）が立って初めて LTV>CAC（広告費の回収）が成立し、その先に課金がある。→ **first_session（初回で刺さる→翌日また来る）が最優先の採点軸**。
- **採点しないもの**: 「クリップ映え・外部拡散力（virality）」は採点軸から外す（発信力が無い前提で最適化する意味がない）。Roblox推薦に乗る前提のオーガニック設計も主軸にしない。
- 広告クリエイティブ（サムネ/動画）と実プレイ体験を一致させる（看板に偽りなし）。広告で過大な期待を作ると初回離脱が増え逆効果。固有名（短く口に出せる名前）は広告の想起・再検索に効くので“あれば良い”が、それ単体を点数化はしない。
- social（ゲーム内で一緒に盛り上がる/招待＝既存プレイヤー経由の口コミ）は引き続き別軸として設計する。

## 7. マネタイズ 〔→ monetization〕
- **消耗品 = Developer Product**（時短・ブースト・蘇生・ガチャ・再挑戦）。**買い切り = Game Pass**（永続強化・枠拡張・見た目）。使い分けを明確に。
- **課金動機とタイミングが自然か**。「劣勢/挑戦直前/撃破直後の高揚」など、買いたくなる瞬間が体験の中にあるか。
- **見た目・自己表現課金は強い**（Robloxのアバター文化と接続）。スキン/エフェクト/限定カラー。
- **Pay-to-Winを避ける**。勝敗が金で決まると評価と寿命を削る。時短・見た目・利便に寄せる。
- **健全な経済**：通貨のsink（使い道）とsource（入手）のバランス。インフレで価値が消えない設計。
- ガチャ的レア収集・期間限定イベントが継続課金を生む（収集欲＋FOMO）。ただし射幸性は年齢層・規約に配慮。

## 8. 実現性 & スコープ（個人開発）〔→ feasibility〕
- **軽い（高得点）**：Part/Model中心、Instance.newワールド生成、標準アバター操作、SurfaceGui/BillboardGui、RemoteEvent、DataStore永続化、TweenService、ProximityPrompt、Soundアセット、ゲームパス/開発者プロダクト課金。
- **ちょっと頑張れば**：自前当たり判定、簡易マッチメイク、procedural生成、軽い対戦同期、非UGCの自作アセット。
- **重い（減点・地雷）**：大規模物理同期、多数プレイヤー×巨大物体の物理拘束、本格ネットコード、サーバー権威の複雑な対人、大量の精緻3Dアセット/オリジナルアニメ必須、実行時音声合成（不可）、動くリグ上へのプレイヤー追従。
- **逃がし方の定番**：物理は「見た目だけ物理風＋サーバー権威で位置駆動」、リズムは「クライアント完結＋自動キャリブレーション」、コンテンツは「手作り量産でなく手続き的な組み合わせで深くする」。
- スコープが「個人でちょっと頑張れば」を超えるほど減点。**派手さよりループの完成度**。

## 9. 一貫性 / テーマ / vibe 〔→ vibe, originality〕
- **第一印象で「楽しそう」が伝わる**画・世界観・名前。サムネ訴求はこの軸。
- **サムネ／タグライン／実プレイの一致**。期待と中身がズレると即離脱＆低評価。
- 世界観・配色・トーンの統一。「可愛い」「怖い」「壮大」など狙う感情を1つに絞ってブレさせない。
- **驚き(originality)**：既存ジャンルそのままでなく、ルールの反転・視点転換・意外な題材の掛け合わせを最低1つ。

## 仕様化(idea-to-spec)で各観点を「埋める」チェックリスト
アイデア1本を仕様に落とすとき、上の9観点をこの順で具体値に変換する：
1. **主動詞と30秒ループ** … 入力→結果→報酬の1サイクルを操作レベルで記述
2. **最初の30秒** … 起動〜初成功体験の導線、チュートリアルの有無
3. **進行と数値** … 何を貯め、何が解放され、10分/数時間/数日でどう変わるか（具体的な解放アイテム/ランク）
4. **モバイル操作** … 画面レイアウト、ボタン配置、縦横、最低スペック想定
5. **人数とスケール** … 想定人数、HP/難度/報酬の人数スケール式、ソロ補助、途中参加
6. **初回体験＆集客** … 1回目のプレイでコアの面白さに到達する導線（離脱しない難度・テンポ）、初回セッション終わりに見える「翌日また開く理由(D1フック)」、広告クリエイティブ案(サムネ/動画)と実プレイの一致、固有名
7. **マネタイズ** … 商品リスト（GamePass/DevProduct）、価格帯、購入導線の瞬間
8. **実現性** … 使うRoblox機能、データ構造(DataStoreキー)、同期方針、コンテンツの増やし方、想定工数(S/M/L)
9. **テーマ/vibe** … 世界観・配色・狙う感情・参照イメージ

→ これに「最大リスクと対策」「MVP(最小で出す範囲)とその後」を足すと実装着手できる仕様になる。
